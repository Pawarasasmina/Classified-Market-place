 
# Codex Recovery Rules

- Never delete or rewrite working existing code unless required.
- First inspect the current codebase before making changes.
- Only add or repair missing functionality.
- Preserve existing architecture, naming style, folder structure, API patterns, UI patterns, validation style, and database conventions.
- Do not refactor unrelated working code.
- Do not remove existing features, routes, models, seed data, or UI unless directly required.
